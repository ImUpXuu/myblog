## 评论系统

接入了 Extalk 评论，部署在 Cloudflare 上。

### Extalk 是什么

Extalk 是一个基于 Cloudflare Workers 和 D1 数据库构建的评论系统，专为静态博客设计。

**技术架构：**
- **后端**：Cloudflare Workers（边缘计算）
- **数据库**：Cloudflare D1（SQLite）
- **前端**：轻量级 JS 客户端

**为什么选它：**

1. **免费** - Cloudflare 免费额度完全够用，零成本
2. **速度快** - 全球 275+ 边缘节点，首屏加载 40ms
3. **轻量** - 不拖慢页面，比 Disqus 等轻得多
4. **数据可控** - 数据存在自己账户下，随时可导出
5. **易部署** - 基于 Workers，几分钟就能搞定

**功能：**
- 评论、回复、点赞
- Markdown 支持
- 邮件通知（可选）
- 反垃圾评论
- 浏览量和点赞统计

现在每篇文章底部都有评论框，欢迎来玩。

评论系统的代码已经集成到博客里，每篇文章加载时会自动初始化。

### 浏览量和点赞

每篇文章都有浏览量和点赞数的显示，数据存在评论系统里。

页面切换时会自动刷新数据，不用手动刷新页面。

这个功能是通过调用评论系统的 API 实现的，每次页面加载或切换时都会请求最新数据。

## 移除 51LA

之前用了 51LA 统计，后来发现被投毒了，果断删除。

### 51LA 投毒事件

2026 年 3 月，51LA 统计被曝出存在安全问题：

**问题：**
- 统计代码中嵌入了恶意脚本
- 会收集用户敏感信息（浏览历史、输入内容等）
- 代码经过混淆，难以察觉
- 可能用于中间人攻击或钓鱼

**发现过程：**

某天打开浏览器开发者工具，发现 Network 里有个奇怪的请求，指向一个不明域名。顺着查下去，发现是 51LA 的代码在偷偷加载第三方脚本。

进一步分析发现，这些脚本会：
- 记录用户的键盘输入
- 截取页面内容
- 发送到外部服务器

**处理：**

1. 立即删除了所有 51LA 相关代码
2. 全面排查了其他第三方脚本
3. 更新了所有依赖
4. 决定不再使用任何第三方统计服务

**替代方案：**

现在博客没有接入统计服务，原因：
- 隐私保护：不想收集用户数据
- 性能考虑：少加载一个脚本
- 没必要：看 Vercel 的部署日志就够了

如果真的要统计，考虑：
- Umami（自建，开源）
- Plausible（付费，注重隐私）
- 直接看服务器日志
