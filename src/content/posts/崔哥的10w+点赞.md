# Ping-Cui-GUI

最后更新: 2026.2.8 20:00

**Ping-Cui-GUI** 是一款集高并发性能与智能化绕过策略于一体的 Web 服务压力测试工具。基于 Python 高性能异步架构开发，能够精准模拟海量真实用户访问行为，深度评估服务器在高压环境下的响应极限与稳定性。

## 📊 流量分配模型

90% 核心目录 (banzhuren-8)

10% 基础页面与首页

±10 ID 动态 URL 随机偏移

## 🚀 核心功能矩阵

* **极速并发架构**：支持数百线程同步调度，毫秒级请求响应，全速压测目标带宽。
* **多维反探测机制**：深度伪装浏览器指纹（Sec-Fetch/Ua系列），结合 `agent.txt` 库实现 100+ UA 随机轮换。
* **精准区域模拟**：内置京/沪/冀三大核心 ISP 的 CIDR IP 池，实现应用层地域流量精准伪装。
* **智能熔断节流**：自研退避算法，实时监测 403 (Antibot) 信号，自动实现“压测-降频-恢复”的闭环控制。

## 🛠️ 关键技术深度解析

### 1\. TCP Keep-Alive 隧道复用

利用 `threading.local()` 为每条线程分配持久化 Session。通过复用底层 TCP 隧道，消除了 90% 以上的 DNS 解析与 SSL 握手时间开销，使得单机 QPS 提升 300%-500%。

### 2\. 加权动态 URL 偏移算法

系统采用加权随机概率模型。**90% 的流量**聚焦于 `banzhuren-8` 核心目录（以 11065 为中心，±10 ID 随机波动），**10% 的流量**用于访问首页及校园动态。这种“主次分明”的流量分布能完美模拟真实用户的搜索与点击路径。

### 3\. 动态身份隔离策略

在维持 TCP 连接复用的同时，强制执行 `session.cookies.clear()`。这使得服务器 WAF 观察到的流量表现为：来自同一高带宽网关，但由上万个独立新设备产生的并发请求，有效瓦解基于 Cookie 追踪的防御策略。

### 4\. 呼吸式呼吸退避算法

引入失败状态机。当 403 触发率超过阈值时，线程进入“休眠态”并线性增加延时（最高 5s）；检测到 200 OK 后，逐步释放压力。这种动态频率调整极大延长了测试 IP 的存活周期。
